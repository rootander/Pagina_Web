<h1>Metodología</h1>
<p>La metodología explica cómo se desarrolló el sistema de gestión de inventario con reconocimiento facial para NovaSport S.A.C. Se trabajó con un enfoque práctico y orientado a resolver los problemas reales que tiene la empresa en el control de productos y en la seguridad del acceso.</p>
<h1>1. Tipo de investigación</h1>
<ul>
<li>Aplicada: porque se creó un sistema para solucionar un problema real</li>
<li>Descriptiva: porque se analiza y describe cómo funciona el inventario actualmente</li>
<li>No experimental: porque no se modifican variables, solo se observa la situación como es</li>
<li>Cuantitativa: porque se maneja información numérica del inventario y registros del personal</li>
</ul>
<h1>2. Diseño de la investigación</h1>
<ul>
<li>Documental: se revisó información sobre inventarios, digitalización y reconocimiento facial</li>
<li>Tecnológico: porque el objetivo es construir un sistema que funcione de manera real</li>
</ul>
<h1>3. Técnicas de recolección de información</h1>
<ul>
<li>Observación del proceso actual de inventario</li>
<li>Entrevistas al personal encargado</li>
<li>Revisión de información relacionada con el tema</li>
<li>Análisis de requisitos para definir las funciones del sistema</li>
</ul>
<h1>4. Instrumentos</h1>
<ul>
<li>Guías de entrevista</li>
<li>Notas de observación</li>
<li>Historias de usuario y casos de uso</li>
<li>Diagramas UML</li>
<li>Prototipos simples de las interfaces</li>
<li>Documentación técnica de las herramientas utilizadas</li>
</ul>
<h1>5. Materiales y herramientas</h1>
<ul>
<li>Computadora de desarrollo</li>
<li>Spring Boot para el backend</li>
<li>HTML, CSS y JavaScript para el frontend</li>
<li>MySQL para la base de datos</li>
<li>Python y OpenCV/Face Recognition para el módulo facial</li>
<li>GitHub para control de versiones</li>
<li>EchoAPI para pruebas de API</li>
<li>VS Code como editor</li>
</ul>
<h1>6. Procedimiento</h1>
<ol>
<li>Identificar los problemas en el inventario y en el acceso al sistema</li>
<li>Recoger información mediante entrevistas y observación</li>
<li>Definir los requisitos del sistema</li>
<li>Diseñar la base de datos, los diagramas y las interfaces</li>
<li>Desarrollar el backend con Spring Boot</li>
<li>Desarrollar el frotend con HTML,CSS y JavaScript</li>
<li>Implementar el reconocimiento facial en Python</li>
<li>Integrar ambos módulos</li>
<li>Realizar pruebas y corregir errores</li>
<li>Documentar el funcionamiento del sistema</li>
</ol>
